# Roadmap — BotFatir

## Фаза 1 — MVP (текущая) ✅

- [x] Структура проекта, конфиг, контекст
- [x] Модель объявления + SQLite дедупликация
- [x] Геозона (полигон) + фильтры (комнаты, этаж, цена)
- [x] Скрапер Циан (внутренний API)
- [x] Скрапер Авито (web API)
- [x] Скрапер Домклик (offers-service API)
- [x] Telegram-бот: уведомления + /status /check /pause
- [x] Планировщик опроса каждые 10 мин

## Фаза 2 — Стабильность

- [ ] Деплой на VPS (systemd + docker-compose)
- [ ] Проверка работы с NL IP; при блокировке — RU-прокси
- [ ] Retry + экспоненциальная задержка при ошибках
- [ ] Логирование в файл + ротация
- [ ] Healthcheck: алерт в Telegram если опрос N раз подряд упал
- [ ] Playwright fallback если API отдаёт капчу

## Фаза 3 — Качество выдачи

- [ ] Дедупликация **между площадками** (одна квартира на Циане и Авито)
- [ ] Уточнение полигона по обратной связи
- [ ] Исключение новостроек / только вторичка (опция)
- [ ] Фильтр «только от собственника»
- [ ] Минимальная площадь кухни / общая площадь

## Фаза 4 — UX бота

- [ ] Настройка фильтров через бота (цена, комнаты)
- [ ] Кнопки «Скрыть» / «В избранное» под объявлением
- [ ] Ежедневная сводка: сколько новых за день
- [ ] Экспорт избранного в CSV

## Фаза 5 — Опционально

- [ ] Яндекс.Недвижимость как 4-й источник
- [ ] Оценка «цена ниже рынка» по медиане района
- [ ] Web-дашборд с картой объявлений

## Деплой на VPS (кратко)

```bash
git clone ... && cd BotFatir
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить токены
python -m botfatir
```

Systemd unit `botfatir.service` — см. README после фазы 2.
